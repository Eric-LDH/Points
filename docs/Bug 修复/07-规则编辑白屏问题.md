# 🐛 规则编辑页面白屏问题修复

## 问题描述
用户报告进入积分规则管理的新增/修改功能后页面白屏，无任何功能可以操作。

## 排查过程

### 1. ✅ 检查路由配置
- 路由 `/rule/edit` 已正确配置
- 组件路径正确：`@/pages/rule/edit.vue`
- 路由参数传递正常

### 2. ✅ 检查编译错误
- TypeScript 编译无错误
- Vue 模板语法无错误
- 所有导入语句正确

### 3. ✅ 检查全局样式
- CSS 变量定义完整
- main.scss 文件正常加载
- 样式类名使用正确

### 4. ✅ 检查 Store 初始化
- Pinia store 正确创建
- initDefaultData() 在 App.vue 中调用
- 数据持久化正常

### 5. 🔧 发现问题并修复

**问题根源：**
- 路由参数处理方式可能导致未定义访问
- 缺少详细的错误日志
- 表单初始值 points 为 0 可能误导用户

**修复方案：**

#### a) 安全处理路由参数
```typescript
// 修改前
const ruleId = route.query.id as string | undefined
const isEdit = !!ruleId

// 修改后
const ruleId = computed(() => route.query.id as string | undefined)
const isEdit = computed(() => !!ruleId.value)
```

使用 `computed` 确保响应式更新和安全的空值处理。

#### b) 增强错误处理和日志
```typescript
onMounted(() => {
  console.log('RuleEdit mounted', {
    isEdit: isEdit.value,
    ruleId: ruleId.value,
    rulesCount: store.rules.length
  })
  
  if (isEdit.value && !editingRule.value) {
    console.warn('Edit mode but rule not found:', ruleId.value)
    alert('未找到该规则，请确认规则是否存在')
    router.back()
  }
})
```

#### c) 优化表单默认值
```typescript
const formData = ref({
  name: '',
  category: '' as Rule['category'],
  type: 'reward' as Rule['type'],
  points: 2, // 从 0 改为 2，更友好
  description: '',
  enabled: true,
  sortOrder: 999
})
```
