## 1. 首页连续打卡天数 - 按孩子统计

### 问题描述
切换孩子后，首页的"连续打卡 N 天"仍显示原孩子的数据，没有根据当前选中的孩子重新计算。

### 修复方案

**修改文件**: [src/pages/index/index.vue](file://d:\Project\Points\src\pages\index\index.vue#L168-L209)

**修改前**:
```typescript

  // ... 计算逻辑
})
```

**修改后**:
```typescript
const streakDays = computed(() => {
  // ✅ 先过滤出当前孩子的记录
  const childRecords = pointsRecords.value.filter(r => r.childId === store.currentChildId)
  
  if (childRecords.length === 0) return 0
  
  // 获取所有有记录的日期（去重）
  const dates = [...new Set(childRecords.map(r => r.date))]
  // ... 计算逻辑

```

**关键改进**:
- ✅ 在计算前先通过 `childId` 过滤记录
- ✅ 只统计当前孩子的打卡数据
- ✅ 响应式监听 `currentChildId` 变化

## 💡 核心技术要点

### 1. 响应式数据过滤

**关键代码**:
```typescript
const childRecords = computed(() => 
  pointsRecords.value.filter(r => r.childId === store.currentChildId)
)
```

**原理**:
- ✅ 使用 `computed` 创建响应式计算属性
- ✅ 依赖 `pointsRecords.value` 和 `store.currentChildId`
- ✅ 当任一依赖变化时自动重新计算

### 2. 多条件过滤

**应用场景**:
- 统计本周数据时同时过滤孩子和日期
- 统计分类时同时过滤孩子和规则

**代码示例**:
```typescript
// 先过滤孩子
const childRecords = pointsRecords.value.filter(r => r.childId === childId)

// 再过滤日期
const dayRecords = childRecords.filter(r => r.date === dateStr)
```

### 3. 深色模式变量系统

**CSS 变量定义**:
```scss
:root {
  // 浅色模式变量
  --bg-color: #F9FAFB;
  --card-bg: #FFFFFF;
  --text-primary: #111827;
  
  // 深色模式变量
  --dark-bg: #1F2937;
  --dark-card-bg: #374151;
  --dark-text-primary: #F9FAFB;
}

html.dark {
  // 通过类名覆盖变量
  background-color: var(--dark-bg);
  color: var(--dark-text-primary);
}
```

**使用方式**:
```scss
.card {
  // 使用变量，自动适配深色模式
  background-color: var(--card-bg);
  color: var(--text-primary);
}
```

## ⚠️ 注意事项

### 1. 性能考虑

**优化建议**:
- ✅ 使用 `computed` 而非 `methods`，利用缓存机制
- ✅ 避免在模板中直接进行复杂过滤操作
- ✅ 大数据量时可考虑使用虚拟滚动

### 2. 数据一致性

**确保措施**:
- ✅ 所有统计数据都基于 `pointsRecords.value`
- ✅ 通过 `childId` 过滤保证数据准确性
- ✅ 响应式更新确保数据实时性

### 3. 用户体验

**改进点**:
- ✅ 切换孩子后立即刷新所有相关数据
- ✅ 提供明确的视觉反馈（加载中、空状态等）
- ✅ 深色模式过渡自然流畅
