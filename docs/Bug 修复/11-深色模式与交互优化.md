# 深色模式与交互优化修复说明

## 修复时间
2026-03-23

## 修复的问题

### ✅ 问题 1：状态栏背景色不一致（全面屏支持）

**问题描述**：切换为深色模式后，手机上方听筒位置的背景色与页面背景色不一致。

**原因分析**：
- Android 状态栏虽然设置为透明，但没有完全延伸到听筒区域
- `windowLayoutInDisplayCutoutMode` 设置为 `shortEdges` 而不是 `always`

**解决方案**：

1. **修改 `android/app/src/main/res/values/styles.xml`**：
   - 将 `android:windowLayoutInDisplayCutoutMode` 改为 `always`
   - 添加 `android:windowLightStatusBar` 属性以适配状态栏图标颜色
   - 更新注释说明状态栏和导航栏完全透明

2. **修改 `android/app/src/main/AndroidManifest.xml`**：
   - 添加存储权限声明（为问题 3 做准备）

3. **修改 `src/assets/main.scss`**：
   - 在 `html, body` 中添加 `@media (prefers-color-scheme: dark)` 规则
   - 确保 HTML 元素在系统深色模式下也能正确显示

**修改文件**：
- `android/app/src/main/res/values/styles.xml`
- `android/app/src/main/AndroidManifest.xml`
- `src/assets/main.scss`

### ✅ 问题 2：深色模式偏好未持久化

**问题描述**：深色模式切换后，程序被杀掉再打开时，没有按照上次选中的深色风格显示。

**原因分析**：
- `toggleDarkMode()` 方法没有将设置保存到 localStorage
- 应用启动时只检测系统模式，没有读取用户的历史选择

**解决方案**：

1. **修改 `src/stores/index.ts`**：
   - 在 `loadFromStorage()` 方法中添加从 localStorage 读取深色模式设置
   - 在 `toggleDarkMode()` 方法中将设置保存到 localStorage
   - 确保加载时自动应用对应的 DOM class

2. **修改 `src/App.vue`**：
   - 移除系统深色模式的自动检测代码
   - 确保用户的选择始终优先于系统设置

**数据流**：
```
用户点击切换 → toggleDarkMode() 
  → 更新 darkMode.value 
  → 修改 DOM class 
  → 保存到 localStorage
  
应用启动 → loadFromStorage()
  → 从 localStorage 读取
  → 恢复 darkMode.value
  → 应用对应的 DOM class
```

**修改文件**：
- `src/stores/index.ts`
- `src/App.vue`

### ✅ 问题 3：手动备份无反应（权限问题）

**问题描述**：安装到手机后的 APP，点击手动备份没有任何反应。

**原因分析**：
- Android 10+ 对文件访问权限有严格限制
- 原来的浏览器下载方式 (`a.download`) 在 WebView 中不起作用
- 需要原生能力来分享文件

**解决方案**：

1. **安装 Capacitor Share 插件**：
   ```bash
   npm install @capacitor/share
   npx cap sync android
   ```

2. **修改 `src/stores/index.ts`**：
   - 重写 `exportBackup()` 方法使用 Share 插件
   - 添加 `blobToBase64()` 辅助函数
   - 通过 Share 插件分享 JSON 文本内容

3. **修改 `src/pages/mine/mine.vue`**：
   - 导入 `Share` 插件
   - 更新 `exportBackup()` 方法使用 Share 插件
   - 简化代码逻辑，直接分享 JSON 文本

4. **权限配置**：
   - 在 `AndroidManifest.xml` 中添加存储权限声明
   - `file_paths.xml` 已配置好缓存路径访问权限

**使用方式**：
- 用户点击"手动备份"后，会弹出系统分享菜单
- 可以选择分享到微信、QQ、邮件或其他应用
- 备份内容为 JSON 格式的完整数据

**修改文件**：
- `package.json` (添加 @capacitor/share 依赖)
- `src/stores/index.ts`
- `src/pages/mine/mine.vue`
- `android/app/src/main/AndroidManifest.xml`

### ✅ 问题 4：点击出现蓝色矩形框

**问题描述**：所有点击功能（底部导航栏、我的→积分规则管理按钮等），每次点击后都会出现蓝色背景的矩形框，点击功能执行完后消失。

**原因分析**：
- WebKit 内核默认的点击高亮效果
- `-webkit-tap-highlight-color` 属性未设置

**解决方案**：

1. **修改 `src/assets/main.scss`**：
   - 在全局重置样式 `*` 中添加 `-webkit-tap-highlight-color: transparent`
   - 移除所有点击高亮效果

**修改文件**：
- `src/assets/main.scss`

## 注意事项

1. **Share 插件使用**：
   - Share 插件分享的是文本内容，不是实际的文件
   - 接收方收到的是 JSON 文本，可以保存为 .json 文件
   - 如果需要分享实际文件，需要使用 Filesystem 插件

2. **权限说明**：
   - 添加了存储权限声明（仅用于 Android 10 及以下版本）
   - Android 11+ 使用 scoped storage，不需要额外权限

3. **深色模式持久化**：
   - 用户的选择始终优先于系统设置
   - 不再自动检测系统深色模式
   - 首次使用时默认为浅色模式

## 技术要点

### 1. 全面屏适配
- `windowLayoutInDisplayCutoutMode`: 控制内容是否延伸到刘海/挖孔区域
  - `default`: 默认行为
  - `shortEdges`: 只在短边延伸
  - `always`: 总是延伸（推荐）
- `fitsSystemWindows`: 设为 false 避免自动添加内边距
- `statusBarColor`: 设为透明实现沉浸式状态栏

### 2. localStorage 持久化
- 使用 `localStorage.setItem()` 和 `getItem()` 存储设置
- 使用 `JSON.stringify()` 和 `JSON.parse()` 序列化布尔值
- 在 Pinia store 的 actions 中同时更新状态和持久化

### 3. Capacitor Share 插件
- 使用 `Share.share()` 方法分享内容
- `text` 参数传递 JSON 字符串
- `dialogTitle` 设置分享对话框标题
- 自动调用原生分享界面，无需额外权限

### 4. 移除点击高亮
- `-webkit-tap-highlight-color: transparent` 全局应用
- 不影响实际的点击事件处理
- 提升用户体验，避免视觉闪烁
