# 深色模式修复说明

## 🐛 问题描述

在深色模式下，`showConfirm` 和 `showToast` 组件的背景颜色仍然显示为白色/浅色，与深色模式不协调。

---

## ✅ 解决方案

### 1. ConfirmDialog 组件修复

#### 修改文件
- `src/components/ConfirmDialog.vue`
- `src/assets/main.scss`

#### 修复内容

**ConfirmDialog.vue** - 添加了嵌套的样式选择器，确保文字颜色正确应用 CSS 变量：

```scss
.confirm-modal {
  background: var(--card-bg);
  
  &__title {
    color: var(--text-primary);
  }
  
  &__content {
    color: var(--text-secondary);
  }
  
  // 系统深色模式
  @media (prefers-color-scheme: dark) {
    background: var(--dark-card-bg);
    
    .confirm-modal__title {
      color: var(--dark-text-primary);
    }
    
    .confirm-modal__content {
      color: var(--dark-text-secondary);
    }
  }
  
  // .dark 类名触发深色模式
  .app-container.dark & {
    background: var(--dark-card-bg);
    
    .confirm-modal__title {
      color: var(--dark-text-primary);
    }
    
    .confirm-modal__content {
      color: var(--dark-text-secondary);
    }
  }
}
```

**main.scss** - 在全局样式中添加深色模式覆盖样式：

```scss
html.dark,
.app-container.dark {
  // ✅ ConfirmDialog 组件深色模式
  .confirm-overlay {
    .confirm-modal {
      background: var(--dark-card-bg);
      box-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
      
      &__title {
        color: var(--dark-text-primary);
      }
      
      &__content {
        color: var(--dark-text-secondary);
      }
      
      &__icon {
        &--warning,
        &--danger,
        &--info {
          filter: none;
        }
      }
    }
  }
}
```

### 2. Toast 组件优化

#### 修改文件
- `src/components/Toast.vue`

#### 修复内容

Toast 组件本身已经使用了半透明黑色背景 (`rgba(0, 0, 0, 0.85)`),在深色模式下也能正常显示。

主要优化：
- 保持了原有的渐变背景色（success/error/warning/info）
- 这些颜色在深色背景下更加醒目
- 增强了阴影效果以适配深色模式

```scss
.toast {
  background: rgba(0, 0, 0, 0.85);
  color: white;
  
  &--success {
    background: linear-gradient(135deg, #10B981, #059669);
  }
  
  &--error {
    background: linear-gradient(135deg, #EF4444, #DC2626);
  }
  
  &--warning {
    background: linear-gradient(135deg, #F59E0B, #D97706);
  }
  
  &--info {
    background: linear-gradient(135deg, #6366F1, #4F46E5);
  }
}
```

## 🔍 技术要点

### 1. CSS 变量使用
使用 CSS 变量实现主题切换：
- `--card-bg` / `--dark-card-bg`
- `--text-primary` / `--dark-text-primary`
- `--text-secondary` / `--dark-text-secondary`

### 2. 双重深色模式支持
同时支持两种触发方式：
- **系统偏好**: `@media (prefers-color-scheme: dark)`
- **手动切换**: `.app-container.dark` 类名

### 3. Scoped CSS 穿透
由于 Vue 的 scoped CSS 限制，在全局样式文件 `main.scss` 中添加深色模式样式，确保样式能够正确应用到组件上。

### 4. 嵌套选择器
在组件内部使用嵌套选择器，确保子元素的样式能够继承父元素的深色模式状态：

```scss
.confirm-modal {
  // 浅色模式
  
  @media (prefers-color-scheme: dark) {
    // 系统深色模式
  }
  
  .app-container.dark & {
    // 手动切换深色模式
  }
}
```

## 🎯 最佳实践

### ✅ 推荐做法
1. 使用 CSS 变量定义颜色
2. 同时支持系统和手动深色模式
3. 在全局样式文件中添加组件深色模式样式
4. 使用嵌套选择器确保样式优先级

### ❌ 避免做法
1. 不要使用硬编码的颜色值
2. 不要只依赖一种深色模式触发方式
3. 不要在 scoped 样式中定义可能被覆盖的样式
