### 2. 今日已完成 - 显示惩罚类型记录

**需求描述**: 今日已完成中需要显示惩罚类型的负分记录。

**问题分析**: 
- 之前 `completedRecords` 只过滤 `points > 0` 的记录
- 惩罚类型的负分记录被过滤掉，不显示

**修改内容**:
```typescript
// 之前
const completedRecords = computed(() => {
  return todayRecords.value.filter(r => r.points > 0)
})

// 现在
const completedRecords = computed(() => {
  // 显示今天的所有记录，包括奖励（正分）和惩罚（负分）
  return todayRecords.value
})
```

```vue
>
  {{ record.points > 0 ? '+' : '' }}{{ record.points }}
</div>
```

### 3. 单日补录 - 防止重复补录

**需求描述**: 在单日补录选择规则时，需要判断该日是否已经存在已完成的项，如果存在则不允许再次补录。

**实现方案**:

#### 3.1 UI 层面 - 视觉标识
```vue
<div 
  v-for="rule in enabledRules" 
  :key="rule.id"
  class="rule-card"
  :class="{ 
    selected: selectedRules.includes(rule.id),
    disabled: isRuleCompletedOnDate(rule.id, selectedDate)
  }"
>
  <!-- 已完成的规则显示"已完成"标签 -->
  <div v-if="isRuleCompletedOnDate(rule.id, selectedDate)" class="completed-flag">
    已完成
  </div>
</div>
```

#### 3.2 交互层面 - 点击防护
```typescript
const toggleRule = (rule: Rule) => {
  // 如果该规则在今天已经完成，不允许选择
  if (isRuleCompletedOnDate(rule.id, selectedDate.value)) {
    alert('该规则在选定的日期已经完成过了，无法重复补录！')
    return
  }
  
  const index = selectedRules.value.indexOf(rule.id)
  if (index === -1) {
    selectedRules.value.push(rule.id)
  } else {
    selectedRules.value.splice(index, 1)
  }
}
```

#### 3.3 提交层面 - 最终检查
```typescript
const submitSingle = () => {
  // ... 其他验证
  
  // 再次检查是否有规则已经存在
  const existingRules = selectedRules.value.filter(ruleId => 
    isRuleCompletedOnDate(ruleId, selectedDate.value)
  )
  
  if (existingRules.length > 0) {
    const ruleNames = existingRules.map(id => {
      const rule = enabledRules.value.find(r => r.id === id)
      return rule ? rule.name : ''
    }).filter(name => name).join('、')
    alert(`以下规则在 ${selectedDate.value} 已经完成，无法重复补录：${ruleNames}`)
    return
  }
  
  // ... 正常提交流程
}
```

#### 3.4 核心检查方法
```typescript
// 检查规则在指定日期是否已完成
const isRuleCompletedOnDate = (ruleId: string, date: string) => {
  return store.todayRecords.some(r => r.ruleId === ruleId && r.date === date)
}
```

**效果展示**:
```
┌─────────────────────────────┐
│ 📝 手工补录                 │
│ 2026.03.12  周四            │
│ ─────────────────────────── │
│ 选择规则                    │
│                             │
│ 🧹 晾衣服          +2 分 ✓  │ ← 未完成的可选
│ 📚 早读 20 分钟   +3 分 ✓  │ ← 未完成的可选
│ 🍽️ 帮忙摆碗筷   +2 分【已完成】│ ← 已完成的禁用
│    （半透明，不可点击）       │
│ ✏️ 练字 15 分钟 +2 分【已完成】│ ← 已完成的禁用
│    （半透明，不可点击）       │
└─────────────────────────────┘
```

**三重防护机制**:
1. ✅ **视觉层**: 已完成的规则显示为半透明，带"已完成"标签
2. ✅ **交互层**: 点击已完成的规则弹出警告
3. ✅ **数据层**: 提交时再次检查，防止绕过前端验证
