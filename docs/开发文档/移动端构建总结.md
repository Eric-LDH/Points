## 🚀 如何使用

### 方式一：一键构建（最简单）

**使用批处理文件**：
```bash
# 双击运行或在终端执行
.\build-android.bat
```

**或使用 PowerShell**：
```powershell
.\build-android.ps1
```

### 方式二：手动分步构建

```bash
# 1. 构建 Web 项目
npm run build

# 2. 同步到 Android
npx cap sync android

# 3. 打开 Android Studio
npx cap open android
```

然后在 Android Studio 中：
1. 等待 Gradle 同步完成
2. 点击 "Run" 按钮运行应用
3. 或选择 Build > Build APK(s) 生成安装包

### 方式三：使用新增的 npm 命令

```bash
# 快速同步到 Android
npm run cap:android

# 完整构建流程
npm run build:android

# 打开 Android Studio
npm run cap:open:android

# 如果已安装 Android SDK，可直接构建 APK
npm run apk:debug
```

## 📱 APK 文件位置

构建成功后，APK 文件位于：
```
android\app\build\outputs\apk\debug\app-debug.apk
```

你可以：
- 📤 通过 USB 传输到手机安装
- 📧 通过邮件发送给自己
- 💬 通过微信/QQ 等工具分享
- 🔧 使用 ADB 安装：`adb install app-debug.apk`

## 🐛 常见问题

### 1. "Unable to launch Android Studio"
**原因**: 未安装或路径未配置  
**解决**: 下载安装 Android Studio 或使用命令行构建

### 2. Gradle 同步失败
**解决**: 
- 检查网络连接
- 删除 `android/.gradle` 后重试
- 重启 Android Studio

### 3. 构建的 APK 无法安装
**解决**: 
- 确保手机已开启"未知来源"安装权限
- 检查 Android 版本兼容性
- 使用 Debug 版本测试

### 4. 应用白屏
**解决**: 
- 确认已执行 `npm run build`
- 运行 `npx cap sync` 同步代码
- 清除应用缓存后重试
