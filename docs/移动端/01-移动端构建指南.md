# 📱 移动端打包指南

本文档介绍如何将 Points 积分管理系统打包成 Android 和 iOS 安装包。

## 🛠️ 环境准备

### Android 开发环境
1. **安装 Android Studio**
   - 下载地址：https://developer.android.com/studio
   - 安装时会自动安装 Android SDK 和 Gradle

2. **配置环境变量**
   - 设置 `ANDROID_HOME` 环境变量指向 Android SDK 目录
   - 将 `%ANDROID_HOME%\platform-tools` 和 `%ANDROID_HOME%\emulator` 添加到 PATH

3. **创建模拟器或连接真机**
   - 在 Android Studio 中创建虚拟设备
   - 或使用 USB 连接真实 Android 设备（需开启 USB 调试模式）

### iOS 开发环境（仅限 macOS）
1. **安装 Xcode**
   - 在 Mac App Store 下载安装 Xcode
   - 安装 Xcode 命令行工具：`xcode-select --install`

2. **安装 CocoaPods**
   ```bash
   sudo gem install cocoapods
   ```

## 📦 打包流程

### 1. 构建 Web 项目
```bash
npm run build
```
这会在 `dist` 目录生成生产环境的静态文件。

### 2. 同步到原生平台
```bash
npx cap sync
```
这会将 `dist` 目录的文件复制到 Android 和 iOS 项目中。

### 3. Android 打包

#### 方法一：使用 Android Studio（推荐）
```bash
npx cap open android
```
这会打开 Android Studio，然后：
1. 等待 Gradle 同步完成
2. 点击 "Run" 按钮（绿色三角形）在模拟器或真机上运行
3. 或选择 `Build > Build Bundle(s) / APK(s) > Build APK(s)` 生成 APK 文件

#### 方法二：命令行构建
```bash
cd android
.\gradlew assembleDebug
```
生成的 APK 位于：`android/app/build/outputs/apk/debug/app-debug.apk`

#### 生成 Release APK（用于发布）
```bash
cd android
.\gradlew assembleRelease
```

**配置签名（发布必需）：**
1. 编辑 `android/app/build.gradle`
2. 配置 signingConfigs
3. 生成签名密钥库：
```bash
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-alias
```

### 4. iOS 打包（仅限 macOS）

#### 方法一：使用 Xcode
```bash
npx cap open ios
```
这会打开 Xcode，然后：
1. 选择开发团队（需要 Apple Developer 账号）
2. 点击 "Run" 按钮在模拟器或真机上运行
3. 或选择 `Product > Archive` 生成 IPA 文件

#### 方法二：使用 CocoaPods
```bash
cd ios
pod install
```
然后用 Xcode 打开 `.xcworkspace` 文件。

## 🚀 快速命令参考

```bash
# 开发调试
npm run dev              # 启动 Vite 开发服务器
npx cap sync            # 同步代码到原生项目
npx cap open android    # 在 Android Studio 中打开
npx cap open ios        # 在 Xcode 中打开（仅 Mac）

# 生产构建
npm run build           # 构建 Web 项目
npx cap sync            # 同步到原生平台
npx cap run android     # 在 Android 设备上运行
npx cap run ios         # 在 iOS 设备上运行（仅 Mac）
```

## 📝 常见问题

### 1. Gradle 同步失败
- 检查网络连接
- 删除 `android/.gradle` 目录后重试
- 确保 Java JDK 版本正确（推荐 JDK 17）

### 2. Android Studio 找不到 SDK
- File > Project Structure > SDK Location
- 设置正确的 Android SDK 路径

### 3. iOS 构建失败
- 确保已安装最新 Xcode
- 运行 `pod repo update` 更新 CocoaPods
- 检查 Xcode 中的开发证书和描述文件

### 4. 应用白屏或无法加载
- 确认已执行 `npm run build`
- 运行 `npx cap sync` 同步最新代码
- 检查 `capacitor.config.ts` 中的 `webDir` 配置

### 5. 热更新需求
如果需要动态更新能力，可以使用 Capacitor 的 Live Updates 功能：
https://capacitorjs.com/docs/guides/live-updates

## 🎯 发布准备

### Android 发布清单
- [ ] 配置应用签名
- [ ] 修改 `android/app/src/main/AndroidManifest.xml` 中的权限
- [ ] 更新应用图标和启动图
- [ ] 配置 `android/app/build.gradle` 中的版本号
- [ ] 生成 Release APK 或 AAB

### iOS 发布清单
- [ ] 配置 Apple Developer 账号
- [ ] 创建 App ID 和证书
- [ ] 配置应用图标和启动图
- [ ] 更新 `ios/App/App/Info.plist`
- [ ] 在 Xcode 中配置 Signing & Capabilities
- [ ] Archive 并上传到 App Store Connect
