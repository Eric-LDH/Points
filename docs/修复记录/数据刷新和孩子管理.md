### 1. 修复 index.ts 的类型错误（58-72 行）

**问题描述**: TypeScript 报错"逗号运算符的左侧未使用，没有任何副作用"

**根本原因**: 
```typescript
// ❌ 错误方式 - 函数声明在 return 之后
return {
  initDefaultData,  // 这里引用了后面才声明的函数
  addRule,
  // ...
}

function initDefaultData() { ... }
```

**修复方案**: 
```typescript
// ✅ 正确方式 - 将函数声明移到 return 之前
function initDefaultData() { ... }
function addRule() { ... }
// ... 其他函数声明

return {
  initDefaultData,
  addRule,
  // ...
}
```

### 2. 添加数据刷新机制

**问题描述**: 
- 底部导航切换后数据不更新
- 统计页面数据不同步
- 各页面数值显示不正确

**解决方案**:

#### 2.1 Store 中添加 refreshData 方法
```typescript
// 从 localStorage 加载数据
function loadFromStorage() {
  rules.value = Storage.rules.getAll()
  rewardItems.value = Storage.rewardItems.getAll()
  pointsRecords.value = Storage.pointsRecords.getAll()
  exchangeRecords.value = Storage.exchangeRecords.getAll()
  children.value = Storage.children.getAll()
  currentChildId.value = Storage.children.getCurrentId()
}

// 立即加载一次数据
loadFromStorage()

// 刷新数据（用于页面切换时重新加载）
function refreshData() {
  loadFromStorage()
}
```

**导出方法**: 在 return 语句中导出 `refreshData`

---

#### 3.2 逻辑层面 - 删除确认
```typescript
const confirmDeleteChild = (child: Child) => {
  if (children.value.length <= 1) {
    alert('至少保留一个孩子！')
    return
  }
  
  const confirmed = confirm(`确定要删除孩子"${child.name}"吗？\n\n删除后该孩子的所有积分记录都将被清空，此操作不可恢复！`)
  
  if (confirmed) {
    // 如果删除的是当前孩子，先切换到其他孩子
    if (child.id === currentChildId.value) {
      const otherChild = children.value.find(c => c.id !== child.id)
      if (otherChild) {
        store.switchChild(otherChild.id)
      }
    }
    
    // 删除孩子
    store.deleteChild(child.id)
    
    // 关闭弹窗
    showChildSwitch.value = false
    
    alert(`已删除孩子：${child.name}`)
  }
}
```

**防护措施**:
- ✅ 至少保留一个孩子
- ✅ 二次确认对话框
- ✅ 明确告知删除后果
- ✅ 自动切换到其他孩子

#### 3.3 Store 层面 - 完整删除
```typescript
function deleteChild(id: string) {
  // 1. 删除孩子的所有积分记录
  const recordsToDelete = pointsRecords.value.filter(r => r.childId === id)
  recordsToDelete.forEach(record => {
    Storage.pointsRecords.delete(record.id)
  })
  pointsRecords.value = pointsRecords.value.filter(r => r.childId !== id)
  
  // 2. 删除孩子的所有兑换记录
  const exchangesToDelete = exchangeRecords.value.filter(r => r.childId === id)
  exchangesToDelete.forEach(record => {
    Storage.exchangeRecords.delete(record.id)
  })
  exchangeRecords.value = exchangeRecords.value.filter(r => r.childId !== id)
  
  // 3. 删除孩子本身
  Storage.children.delete(id)
  children.value = children.value.filter(c => c.id !== id)
}
```

**删除顺序**:
1. 积分记录 → 2. 兑换记录 → 3. 孩子信息

## 🔄 数据刷新流程

### 修复前（❌ 错误）
```
用户点击底部导航
  ↓
路由切换到新页面
  ↓
页面组件加载
  ↓
computed 计算属性执行
  ↓
但数据可能不是最新的
  ↓
❌ 显示旧数据
```

### 修复后（✅ 正确）
```
用户点击底部导航
  ↓
调用 navigateTo()
  ↓
执行 store.refreshData()
  ↓
从 localStorage 重新加载所有数据
  ↓
rules.value = Storage.rules.getAll()
pointsRecords.value = Storage.pointsRecords.getAll()
...
  ↓
路由切换到新页面
  ↓
页面组件加载
  ↓
computed 基于最新数据计算
  ↓
✅ 显示最新数据
```
