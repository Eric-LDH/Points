# 问题修复说明

## 修复的问题列表

### ✅ 问题 1：手机听筒位置背景色不一致（全面屏支持）

**问题描述**：手机顶部听筒区域的背景色与页面背景色不一致，未实现全面屏适配。

**解决方案**：
1. 修改 `android/app/src/main/res/values/styles.xml`：
   - 添加状态栏颜色设置为透明
   - 添加导航栏颜色设置为透明
   - 启用全面屏布局模式（windowLayoutInDisplayCutoutMode）
   - 禁用 fitsSystemWindows 以避免内边距问题

2. 修改 `android/app/src/main/AndroidManifest.xml`：
   - 添加 `windowSoftInputMode="adjustResize"` 以优化键盘弹出时的布局

**修改文件**：
- `android/app/src/main/res/values/styles.xml`
- `android/app/src/main/AndroidManifest.xml`

---

### ✅ 问题 2：深色模式未持久化保存

**问题描述**：切换深色模式后，杀掉应用再重新打开时，没有按照上次选中的深色风格显示。

**解决方案**：
1. 修改 `src/stores/index.ts`：
   - 在 `toggleDarkMode()` 方法中将设置保存到 localStorage
   - 在 `loadFromStorage()` 方法中从 localStorage 读取深色模式设置
   - 应用启动时自动恢复上次的深色模式状态

2. 修改 `src/App.vue`：
   - 移除系统深色模式的自动检测
   - 确保用户的选择始终优先于系统设置

**修改文件**：
- `src/stores/index.ts`
- `src/App.vue`

**数据流**：
```
用户点击切换 → toggleDarkMode() 
  → 更新 darkMode.value 
  → 修改 DOM class 
  → 保存到 localStorage
  
应用启动 → loadFromStorage()
  → 从 localStorage 读取
  → 恢复 darkMode.value
  → 应用对应的 DOM class
```

### ✅ 问题 3：手动备份无反应（权限问题）

**问题描述**：安装到手机后，点击"手动备份"按钮没有任何反应。

**问题根源**：
- 浏览器环境的下载方式（创建 `<a>` 标签点击）在 Capacitor 原生环境中不起作用
- Android 文件系统需要特定的权限和 API

**解决方案**：
1. 修改 `src/utils/storage.ts`：
   - 检测是否在 Capacitor 原生环境
   - 在原生环境中使用 Capacitor Share 插件分享文件
   - 将 Blob 转换为 Base64 格式以供分享
   - 保留浏览器的传统下载方式作为降级方案

2. **需要安装 Share 插件**（重要！）：
   ```bash
   npm install @capacitor/share
   npx cap sync android
   ```

**修改文件**：
- `src/utils/storage.ts`

**使用方式**：
- 点击"手动备份"后，会弹出系统分享对话框
- 可以选择保存到文件管理器、发送到微信/QQ 等

**备选方案**：
如果暂时不想安装 Share 插件，可以临时使用浏览器的下载方式，但需要在 Android 中添加存储权限：
```xml
<!-- android/app/src/main/AndroidManifest.xml -->
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
```

### ✅ 问题 4：点击出现蓝色矩形框

**问题描述**：底部导航栏、我的页面按钮等所有可点击元素，点击时会出现蓝色背景的矩形框，影响视觉体验。

**问题根源**：
- 移动端浏览器默认的点击高亮效果（tap highlight）
- WebKit 内核的 `-webkit-tap-highlight-color` 属性

**解决方案**：
1. 修改 `src/assets/main.scss`：
   - 在全局重置样式中添加 `-webkit-tap-highlight-color: transparent`
   - 同时添加标准属性 `tap-highlight-color: transparent`

**修改文件**：
- `src/assets/main.scss`

**CSS 代码**：
```scss
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
  // 移除移动端点击高亮效果
  -webkit-tap-highlight-color: transparent;
  tap-highlight-color: transparent;
}
```

## ⚠️ 注意事项

1. **Share 插件必须安装**：
   - 如果不安装，备份功能会提示错误
   - 安装后必须执行 `npx cap sync android`

2. **Android 版本兼容性**：
   - 全面屏支持需要 Android 9.0 (API 28) 及以上
   - 项目已设置 minSdkVersion = 24，但全面屏特性在旧版本会自动降级

3. **深色模式优先级**：
   - 用户手动选择 > 系统设置
   - 一旦用户切换过，就会永久保存直到下次手动切换

4. **点击高亮移除的影响**：
   - 所有元素都会移除点击高亮
   - 如果需要保留某些元素的反馈效果，可以使用 CSS active 伪类自定义

## 📝 技术细节

### Capacitor Share 插件工作原理
```typescript
// 1. 将数据转换为 Blob
const blob = new Blob([JSON.stringify(data)], { type: 'application/json' })

// 2. Blob 转 Base64
const base64 = await blobToBase64(blob)

// 3. 调用原生分享 API
await Capacitor.Plugins.Share.share({
  title: '备份数据',
  text: '描述文字',
  files: [{ data: base64, fileName: 'backup.json' }]
})
```

### 全面屏适配原理
```xml
<!-- 状态栏透明 -->
<item name="android:statusBarColor">@android:color/transparent</item>

<!-- 导航栏透明 -->
<item name="android:navigationBarColor">@android:color/transparent</item>

<!-- 允许内容延伸到刘海区域 -->
<item name="android:windowLayoutInDisplayCutoutMode">shortEdges</item>

<!-- 禁用系统 FITS_SYSTEM_WINDOWS，避免自动添加内边距 -->
<item name="android:fitsSystemWindows">false</item>
```
