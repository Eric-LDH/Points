# 🐛 今日已完成项重启后加载失败修复

## 问题描述

**症状**: 每次重启程序（刷新页面或重新启动开发服务器）后，首页的"今日已完成"列表都加载不出来，即使 localStorage 中有数据。

**影响**: 
- ✅ 用户完成的积分记录在刷新后消失
- ✅ 今日积分统计归零
- ✅ 需要重新完成任务才能显示

---

## 根本原因分析

### 问题根源

在 Pinia store 初始化时，存在以下问题：

```typescript
// ❌ 错误的方式（之前）
export const useAppStore = defineStore('app', () => {
  // State - 直接从 localStorage 读取
  const rules = ref<Rule[]>(Storage.rules.getAll())
  const pointsRecords = ref<PointsRecord[]>(Storage.pointsRecords.getAll())
  // ...
  
  // 但是 initDefaultData() 没有重新加载数据
  function initDefaultData() {
    if (children.value.length === 0) {
      // 创建默认孩子
      Storage.children.add(defaultChild)
      children.value = [defaultChild]
      // ❌ 但没有重新从 storage 加载其他数据
    }
  }
})
```

### 问题分析

1. **初始化时机问题**:
   - Store 创建时从 localStorage 读取数据
   - 但如果 localStorage 为空，ref 被初始化为空数组
   - `initDefaultData()` 只创建默认孩子和规则，不重新加载

2. **数据同步问题**:
   - 应用启动时没有调用 `initDefaultData()`
   - 即使 localStorage 有数据，store 的 ref 也可能为空

3. **响应式更新缺失**:
   - 没有确保每次访问 store 时都从 storage 同步最新数据

---

## 解决方案

### 修复策略

采用 **"每次创建 store 实例时都重新加载"** 的策略：

```typescript
// ✅ 正确的方式（现在）
export const useAppStore = defineStore('app', () => {
  // 1. 先初始化为空数组，避免 SSR 水合问题
  const rules = ref<Rule[]>([])
  const pointsRecords = ref<PointsRecord[]>([])
  const children = ref<Child[]>([])
  const currentChildId = ref<string | null>(null)
  
  // 2. 定义加载函数
  function loadFromStorage() {
    rules.value = Storage.rules.getAll()
    rewardItems.value = Storage.rewardItems.getAll()
    pointsRecords.value = Storage.pointsRecords.getAll()
    exchangeRecords.value = Storage.exchangeRecords.getAll()
    children.value = Storage.children.getAll()
    currentChildId.value = Storage.children.getCurrentId()
  }
  
  // 3. 立即加载一次数据
  loadFromStorage()
  
  // 4. 在 main.ts 中调用 initDefaultData()
})
```

### 1. [stores/index.ts](file://d:\Project\Points\src\stores\index.ts) - 状态管理

#### 修改点 1: 重构数据加载逻辑
```typescript
// 之前
const rules = ref<Rule[]>(Storage.rules.getAll())

  pointsRecords.value = Storage.pointsRecords.getAll()
  // ...
```

**优点**:
- ✅ 确保每次创建 store 实例都从 localStorage 同步
- ✅ 避免 SSR 水合不匹配问题
- ✅ 逻辑清晰，易于维护

#### 修改点 2: importBackup 后重新加载
```typescript
async function importBackup(file: File) {
  const data = await Storage.importFromFile(file)
  Storage.restore(data)
  
  // 重新加载数据，确保状态同步
  loadFromStorage()
}
```

**优点**:
- ✅ 导入备份后立即刷新所有状态
- ✅ 避免手动逐个赋值

## 经验总结

### 教训
1. **不要假设 localStorage 始终有数据**
   - 首次运行时 localStorage 为空
   - 需要显式检查和创建默认数据

2. **Store 初始化要谨慎**
   - 直接在 ref 初始化时读取 localStorage 可能导致问题
   - 最好分两步：先创建空 ref，再加载数据

3. **确保数据同步**
   - 任何修改 storage 的操作后都应考虑刷新 store 状态
   - 导入备份等操作后要重新加载

### 最佳实践
1. ✅ **分离关注点**: 数据加载逻辑独立成函数
2. ✅ **明确初始化**: 在应用入口显式调用初始化
3. ✅ **统一数据源**: 始终以 localStorage 为准
4. ✅ **及时同步**: 修改后立即刷新状态
