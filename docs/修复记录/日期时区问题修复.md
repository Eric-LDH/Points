# 日期时区问题修复说明

## 问题描述

在统计页面（本周表现）中，用户在周二完成的任务被错误地统计到了周三。这是因为代码中使用了 `toISOString().split('T')[0]` 来获取日期字符串，该方法会将本地时间转换为 UTC 时间，导致日期偏移。

### 问题原因

当本地时间是周二晚上（例如 20:00），使用 `toISOString()` 会转换为 UTC 时间。如果当地时区是 UTC+8（中国标准时间），那么：
- 本地时间：周二 20:00 (UTC+8)
- UTC 时间：周二 12:00 (UTC+0)

但如果本地时间是周二晚上 20:00 之后，比如 22:00：
- 本地时间：周二 22:00 (UTC+8)  
- UTC 时间：周二 14:00 (UTC+0)

更严重的情况是，如果本地时间是周三凌晨 01:00：
- 本地时间：周三 01:00 (UTC+8)
- UTC 时间：周二 17:00 (UTC+0) ← **日期变成了周二！**

这导致了日期统计错误。

## 修复方案

创建了一个统一的本地时间格式化函数 `formatDateOnly`，直接使用本地时间的年、月、日构建日期字符串，避免时区转换问题：

```typescript
// 将 Date 对象格式化为 YYYY-MM-DD 格式的本地日期字符串（避免时区问题）
const formatDateOnly = (date: Date): string => {
  const year = date.getFullYear()
  const month = (date.getMonth() + 1).toString().padStart(2, '0')
  const day = date.getDate().toString().padStart(2, '0')
  return `${year}-${month}-${day}`
}
```

## 技术要点

1. **避免使用 `toISOString()` 获取日期**：该方法会进行时区转换，不适合用于本地日期比较
2. **使用本地时间方法**：`getFullYear()`、`getMonth()`、`getDate()` 直接获取本地时间组件
3. **统一日期格式化**：所有需要 YYYY-MM-DD 格式的地方都使用 `formatDateOnly` 函数
4. **保持 ISO 8601 时间戳**：`completedAt`、`exchangeDate` 等完整时间戳仍使用 `toISOString()`，因为它们需要包含时区信息
